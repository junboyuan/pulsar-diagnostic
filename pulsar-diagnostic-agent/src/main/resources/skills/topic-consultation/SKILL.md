# 主题咨询技能

## 概述

提供Apache Pulsar主题设计、配置和优化方面的专业咨询。

## 适用场景

在以下情况下使用此技能：
- 设计新主题
- 优化现有主题
- 选择分区策略
- 设置保留策略

## 处理流程

### 1. 了解需求

询问明确的问题：
- 预期吞吐量？
- 生产者/消费者数量？
- 消息大小？
- 保留要求？
- 需要顺序保证吗？

### 2. 分析现有主题（如适用）

对于现有主题：
```
getTopicInfo(topic) → getTopicStats(topic) → getTopicSubscriptions(topic)
```

### 3. 提供建议

```
## 主题咨询报告

### 当前状态（如已存在）
- 主题：[名称]
- 分区数：X
- 吞吐量：X 条/秒
- 订阅数：Y

### 建议

#### 分区策略
- 当前：X个分区
- 建议：Y个分区
- 原因：[解释]

#### 保留策略
- 建议：[基于大小/时间]
- 原因：[解释]

#### 配置建议
- maxMessageSize：X KB
- ttlDuration：Y 小时
- retentionSize：Z GB

#### 最佳实践
1. [实践1]
2. [实践2]

### 命名规范
建议格式：persistent://[租户]/[命名空间]/[域]-[实体]-[事件类型]
```

## 可用工具

| 工具 | 用途 |
|------|------|
| `getTopicInfo` | 获取主题配置 |
| `getTopicStats` | 获取主题统计 |
| `getTopicSubscriptions` | 列出订阅 |
| `listTopicsInNamespace` | 列出所有主题 |

## 决策矩阵

### 分区策略

| 场景 | 建议 |
|----------|----------------|
| 高吞吐量 | 多分区 |
| 严格顺序 | 单分区 |
| 多消费者 | 按键分区 |
| 全局消费者 | 轮询 |

### 保留策略

| 用例 | 建议 |
|----------|----------------|
| 事件流 | 基于时间（7-30天） |
| 日志聚合 | 基于大小（100GB+） |
| 工作队列 | 基于确认删除 |
| 审计追踪 | 无限期保留 |

## 需要避免的常见错误

- 分区太少（瓶颈）
- 分区太多（开销）
- 无保留策略（磁盘满）
- 错误的订阅类型
- 缺少Schema验证
